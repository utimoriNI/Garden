## CursorはREADMEを書くだけで衝撃的

[honeshabri](https://note.com/honeshabri)

参加中

最近、AIエディタのCursorを文章執筆に使うのが良いという記事をよく見かけるようになった。

[**いま文章を書くのに「CURSOR」を使わないのは損だ** *プログラマたちが重宝しているコードエディタ「CURSOR」を文章書くのに使うとやたら便利であると評判だ。生成AIというとC* *ascii.jp*](https://ascii.jp/elem/000/004/253/4253872/)

これまで俺はClaudeのアーティファクト機能を使って執筆していたが、直接編集できないため、一度Obsidianに移してから編集し、再度アップロードするという不便さがあった。

Cursorなら直接エディタで編集できる上に、ChatGPTもClaudeも使える。さらにVS Codeベースでgit管理もできるため、Claudeだけ契約するよりも効率的だと判断して使い始めた。ようやく腰を上げて触れてみたという段階だ。

note用のリポジトリを作成するにあたり、どのような構成にすべきか悩んでいた。ベストプラクティスを知りたくて、そこでCursorのAIに「こういう構成にしたい」という要望を音声入力で伝え、READMEファイルに書いてもらうことにした。AIは「分かりました」と言って、READMEを書き始めた。ここまでは想像の範囲内だった。これまでのClaudeがアーティファクトに記事を書くのと変わらない。

問題はここからである。AIはREADMEを書くだけでなく、いきなり「次に説明した構成に従って必要なフォルダとファイルを作成しましょう」と言って、勝手にフォルダ構造の実装を始めたのだ。

ここから先は有料部分です

![画像](https://assets.st-note.com/img/1741087233-ywlWnjR0YrEbQmMSZCO1GfIv.png?width=1200)

こいつ、勝手に動くぞ

俺がCursorのAIに指示したのは、あくまでも「構成を考えてくれ」ということと、喋った内容をREADMEに書いてくれということだけだった。セットアップを全部やってくれとはお願いしていなかったのに、AIが勝手に「READMEにこういうことを書いたんだから、お前が次にやりたいのはフォルダを作ることだろ」と判断して、自ら行動を起こしたのだ。

「できたな」と思っていたら、AIは次々と作業を進めていった。「お前の要望的にこういうテンプレートが欲しいだろう」と、テンプレートファイルの中身まで勝手に作成し始める。例によってテンプレートの中身を作ってくれとは一言も言っていないのに、だ。

![画像](https://assets.st-note.com/img/1741087286-KhjeDQviL6kuN3s1BCIl9cxZ.png?width=1200)

また勝手に

これまで俺が使ってきたAIはそのサービスの中でほぼ完結していることしか使っていなかった。ChatGPTやClaudeの会話、画像生成なども基本的にはサービス内でのやり取りだ。せいぜい「ファイル保存しますか」というダイアログが出てくる程度で、ネット検索をしても、俺への出力はサービス内に閉じていた。

ところがCursorは完全に新しいファイルを作り始める。ファイルを作って、編集して、しかも指示していないけれどやってほしいことまで先回りして実行するのだ。もちろん、この1年くらいの間に「もう全部AIがやってくれます」みたいな話は見てきた。でも、いざ自分の体験として、こっちが依頼していなくてもやってほしいことまで全部わーっとやってくれると「うわー」ってなる。正直に言って。

この体験から痛感したのは、コンピュータ内で完結する作業は全てAIにやらせるべきだということだ。人間がやるというのは、むしろ愚かなこと。「AIの方が早くて正確なのに、なぜ人間がやろうとしているの？それは無駄でしょ」と言われる時代が来ている。そして人間の役割はAIのために環境を整え、指示を出すことなのだ、と。

今回の衝撃はかなり大きかった。Cursorのようなファイルやディレクトリを直接操作するAIツールを使う体験は、積極的にAIエディタを取り入れるプログラマーと一般ユーザーの間の認識の差を浮き彫りにする。

俺自身、AIやClaudeをnoteに何度も使ってきたが、ここまでの可能性を認識していなかった。今日に至るまで、こんな風に感じたことはなかった。時代は確実に変わっている。ちょうどガンダムを見終わったばかりだから「これがニュータイプか」と言わざるを得ない。

マジでこの手のAIエージェントには早く触れた方がいい。認識が根本から変わるからだ。コンピュータとのインタラクションに対する考え方が、一気に進化するだろう。
