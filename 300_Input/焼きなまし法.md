# [焼きなまし法](https://zellij.hatenablog.com/entry/20120825/p1)

[最適化問題](http://d.hatena.ne.jp/keyword/%BA%C7%C5%AC%B2%BD%CC%E4%C2%EA)を解くための探索[アルゴリズム](http://d.hatena.ne.jp/keyword/%A5%A2%A5%EB%A5%B4%A5%EA%A5%BA%A5%E0)の1つに[焼きなまし法](http://d.hatena.ne.jp/keyword/%BE%C6%A4%AD%A4%CA%A4%DE%A4%B7%CB%A1)というものがある。

[最急降下法](http://d.hatena.ne.jp/keyword/%BA%C7%B5%DE%B9%DF%B2%BC%CB%A1)のように、値が小さくなる方向に少しずつ探索を進めていく[アルゴリズム](http://d.hatena.ne.jp/keyword/%A5%A2%A5%EB%A5%B4%A5%EA%A5%BA%A5%E0)では、その出発点に依存して局所最適解に陥ることが多い。  
その結果として、大域的最適解が求まらないという問題がある。

[![f:id:Zellij:20130614111139p:image](https://cdn-ak.f.st-hatena.com/images/fotolife/Z/Zellij/20130614/20130614111139.png "f:id:Zellij:20130614111139p:image")](http://f.hatena.ne.jp/Zellij/20130614111139)

この問題を解決して、最終的に大域的最適解が求まりやすくなるように改良したのが[焼きなまし法](http://d.hatena.ne.jp/keyword/%BE%C6%A4%AD%A4%CA%A4%DE%A4%B7%CB%A1)。

考え方は単純で、「たまには解から遠ざかる方向にも進んでみよっか」というもの。

常に値が小さくなる方向ではなくて、たまには逆向きに進んでみると、もっとよい解が見つかるのではないか、と考える。

しかしながら、この「たまには」という言葉はあいまいすぎるので、確率pで、という具合に確率の話で説明する必要がある。

[焼きなまし法](http://d.hatena.ne.jp/keyword/%BE%C6%A4%AD%A4%CA%A4%DE%A4%B7%CB%A1)では、この確率pは、値の変化量にも影響を受けるけど（解から大きく遠ざかる方向には、あまり行かないように確率pを小さくする）、もうひとつ、温度パラメータTにも依存するようにする。

温度パラメータって？？

温度パラメータTは、最初は大きく、時間が経つにつれて小さくなるようなパラメータで、これが[焼きなまし法](http://d.hatena.ne.jp/keyword/%BE%C6%A4%AD%A4%CA%A4%DE%A4%B7%CB%A1)の名前の由来となっている（最初は熱いけど、次第に熱が冷めていく）。

温度パラメータTが大きいときは、逆向きに進む確率が大きくなり、温度パラメータTが小さい時は逆向きに進む確率が小さくなるように確率pの式を設定する。

さて、以上をまとめると、[焼きなまし法](http://d.hatena.ne.jp/keyword/%BE%C6%A4%AD%A4%CA%A4%DE%A4%B7%CB%A1)は次のような探索[アルゴリズム](http://d.hatena.ne.jp/keyword/%A5%A2%A5%EB%A5%B4%A5%EA%A5%BA%A5%E0)だと言える。

「値が小さくなる方向に少しずつ探索を進めていくけど、たまには（最初のうちは頻繁に、後半は稀に）解から遠ざかる方向にも進んでみる。」
