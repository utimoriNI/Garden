# リサーチのはじめかた
#### メモ
- p17
	- やりたいことはあっても、正確に言語化できないかもしれない。それどころか、なにをやりたいのか全く気がついてないこともある。
	- それは自分のことがあまりよく分かっていないからだったり、自分の興味があること、好きなことが「やりたいこと」の数に入るとは思っていないからだったりする。
	- 私たちはみな外的な期待に囲まれて生きているから、それを自分自身の期待だといつのまにか思い込む。
	- 自己を省みるすべや自分を信頼するすべを学ぶより、人はもっと手っ取り早い道を選びがちだ。
	- 他人のやりたいことを採用して、それが自分のやりたいことだという振りを一生懸命してしまうのである
- p18 自己中心的研究の定義
- p33 研究の着手段階では最初に自分の内面を見つめ、次に外側に目を向ける
	- 内面に目を向けることで重要でないこと、重要だと思っていること、本当に重要なことを区別する
- p35 やってみよう ここで、いますぐ書く
	- 目標：研究のアイディア出しの段階で、メモをとることを習慣にする。まだプロジェクトがきちんと形をとる前であっても、研究に関して考えたことや推測したこと、目的などを書き留めるところから始めよう
##### 第1章 問いとは？
- p40 範囲が広くて漠然としたテーマをどのようにして具体的で興味をそそる問いに落とし込むか
	- テーマはあるに超したことはないが、完全に頼ることはできない
		- 範囲が広く、抽象的すぎる
- テーマ
	- 名詞
	- 好奇心の対象を示す
	- 無数の問いが生じるが、その問いは千差万別で方向性が定まらない
	- 答えがない
- 問い
	- クエスチョンマークで終わる文章
	- 好奇心の対象を示すと同時に、その好奇心をどう満たしたいかという意向を示す
	- より具体的で、関連する問いを生み出す
	- 答えがある、時にはいくつも
- p48 テーマ問いに落とし込むために
	- 自分を飾らない
		- 問いを探す段階では、問いを高度にしようとせず、稚拙な言葉で表現して良い
	- 批判せず、肯定的に受け止める
		- 支離滅裂で偏った問いでも出てこないよりはマシ
	- アイディアを書き留める
		- 考えるだけでは不十分、それを文字に起こして思考の跡を残しておくことが必要
		- 他の目的に利用出来ることもある
	- 問いは内側から生み出すこと
		- 目指すべきは自分の知識や好奇心によって問いを生み出していくこと
		- 問いを生み出す際に、架空の審判に気に入られるかを気にしてはいけない
	- テーマに興味があることを表明するのではなく、テーマに対する疑問を説明してみる
- p53 やってみよう 自分自身を検索する
	- 目標：一次資料の検索結果を用いて、そのテーマのうち自分にとって最も関心のある側面を把握し、その関心に基づいて問いを作成する
- p64 やってみよう 退屈を手がかりにする
- p82 問いを問題に昇華させる
- p83 問題とでたらめな好奇心の集まりを見分ける方法
- p87やってみよう 問いに診断テストを実行する
- p95 やってみよう 一次資料を使って問いを鍛える
- p107 やってみよう 思い込みを可視化する
- p115 やってみよう 問いと問いを結びつける問題を特定する
- p122 一次資料の定義
	- イラストにおける一次資料はpsdファイルの肌レイヤーになりそう
