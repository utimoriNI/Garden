- # 開発日誌 — Raindrop / Obsidian 統合検索

日付: 2026-07-11

## 概要

Raindrop に保存した情報に加え、GitHub 上で管理している Obsidian Vault もセマンティック検索できるようにするための設計・実装を進めた。

Obsidian はローカル常駐プロセスではなく、GitHub App を通じて Vercel 上のアプリが同期する構成を採用した。検索対象は `🎁Topic/` で始まるタグが付いた Markdown ノートのみに限定する。

## 決定事項

- GitHub App に対象 Vault リポジトリの `Contents: Read-only` 権限のみを付与する。
- Obsidian ノートは GitHub API から取得し、Vercel / Turso 上の検索インデックスに保存する。
- `🎁Topic/` タグは検索時のフィルタではなく、同期時の取り込み条件とする。
- タグを外したノート・削除されたノートは、次回の正常同期時に検索インデックスから削除する。
- Raindrop と Obsidian の結果は統合表示し、source・tag・folder で絞り込めるようにする。
- GitHub Push Webhook は署名検証して同期予約を記録し、定期同期および管理画面からの手動同期で反映する。

## 実装内容

- Obsidian Markdown の frontmatter と本文中のタグを解析する処理を追加した。
- GitHub App の JWT と Installation Token を使い、指定リポジトリ・ブランチの Markdown を取得する処理を追加した。
- `documents`、`document_chunks`、`document_fts` の共通ドキュメント索引を追加した。
- `🎁Topic/` 配下のタグを持つノートだけを embedding・全文検索の対象にした。
- `/api/sync/obsidian` に Obsidian 同期 API を追加した。
- `/api/github/webhook` に GitHub Push Webhook の署名検証と同期予約処理を追加した。
- 検索 UI に source と Obsidian folder のフィルタを追加した。
- MCP に `search_personal_knowledge` を追加し、旧 `search_raindrops` は互換機能として維持した。
- Admin 画面に Obsidian 同期ボタンとノート数表示を追加した。
- README と `.env.example` に GitHub App / Webhook / Vercel の設定手順を追加した。

## 検証

以下を実行し、Raindrop と Obsidian の両方で検索できることを確認した。

- `npm run verify`
- `npm run build`

Obsidian のスモークテストでは、`🎁Topic/AI` タグと `Topics` フォルダを指定した検索で対象ノートが返ることを確認した。

## 次の作業

GitHub App の作成・対象 Vault へのインストール、Vercel 環境変数と Webhook の設定を行い、実際の Vault を使った初回同期と Push 反映を確認する。

# 情報管理・検索・タスク管理の運用検討メモ

## 背景

現在、Codexを使って、Raindrop内に保存してある情報をセマンティック検索できるアプリケーションを作っている。

その検索対象に、Obsidian内のドキュメントも含めたいと考えている。

例えば、出かけるスポットを調べるときに、Webで保存した情報だけでなく、Obsidianでまとめた情報や整理した情報も検索対象に含められると便利そうだと感じた。

## 課題：RaindropとObsidianに情報が分散している

現状、情報が以下の2つに分散している。

- Raindrop
- Obsidian

そのため、両方をどのように検索対象として扱うべきか迷っている。

## Raindropのメリット

Raindropは、Web上の情報を保存する用途に向いている。

特に以下の点が強い。

- 保存がしやすい
- Webクリップがしやすい
- Web上の情報を蓄積する用途では最適に近い

そのため、Webページや記事などを保存する入口としては、Raindropが適している。

## Obsidianのメリット

Obsidianは、情報の整理に向いている。

Codexと併用することで、以下のような使い方ができる。

- 散歩ルートをAIに考えてもらう
- 個人的に整理した情報を蓄積する
- 複数の情報を集約して、自分用に再整理する
- 出かけるスポットや調査内容を自分の文脈でまとめる

単なるWeb保存ではなく、自分の考えや整理済みの情報を扱う場所として便利。

## 検討したいこと

RaindropとObsidianの両方を検索できるようにしたい。

ただし、以下の点で構築方針に迷っている。

- RaindropとObsidianを別々に検索するのか
- 両方の情報を一つの検索インデックスに統合するのか
- Obsidian側のMarkdownファイルをどのように取り込むのか
- RaindropのWebクリップ情報とObsidianの整理済みメモを、検索結果上でどう区別するのか
- どちらを情報の一次保存場所にするのか
- どちらを整理・再編集の場所にするのか

今後、RaindropとObsidianを横断検索するための構成を検討したい。

# 日記メモの運用改善

## 現状

現在は、日記を1日単位で管理している。

ただし、今後は以下のような単位で日記を管理することも検討している。

- 平日5日間で1つ
- 土日で1つ

## 課題

現状の運用では、週単位や複数日単位で日記を書いたとしても、その日の日記として認識されてしまう。

例えば、「今週分の日記」として作成したメモでも、システム上は作成日やファイル名に紐づいて、その日の日記のように扱われてしまう。

## 今後検討したいこと

日記の単位を、1日単位から複数日単位へ柔軟に変えられるようにしたい。

検討ポイントは以下。

- 平日用の日記メモ
- 週末用の日記メモ
- 週単位の日記メモ
- 1日単位の日記との使い分け
- Obsidian上でのファイル名やテンプレートの設計
- 日付ベースで自動認識される仕組みとの整合性

# タスク管理ツールの検討

## 現状の候補

タスク管理について、以下の候補で迷っている。

- Todoist
- Notion

## Todoistの印象

Todoistは便利そうに見える。

特に、Codexなど複数の経路から操作できそうな点がメリット。

メリットは以下。

- タスク管理専用ツールとして使いやすそう
- Codexなどから操作しやすそう
- 外部連携の選択肢が多そう
- チャット経由でタスク登録しやすそう

一方で、ツールを増やしたくないという懸念がある。

## Notionの印象

Notionもタスク管理に使えそう。

外部からアクセスしたり操作したりすることは可能そうだが、実現するには追加の準備が必要になる可能性がある。

想定される課題は以下。

- iOSショートカットを作成する必要があるかもしれない
- 自分で連携用のツールを準備する必要があるかもしれない
- Todoistよりもタスク管理専用としての使いやすさは検証が必要
- CodexやChatGPTからの操作性を確認する必要がある

## 今後検証したいこと

TodoistとNotionのどちらが自分の運用に合うか検証したい。

検討ポイントは以下。

- Codexからタスクを登録できるか
- ChatGPTからタスクを確認・登録できるか
- iOS、Mac、Windowsなど複数環境で使いやすいか
- ツールを増やすデメリットを許容できるか
- Notionで実現する場合に必要な実装・設定の手間
- Todoistで無料プランの範囲内でどこまで実現できるか
- 日記やObsidianの情報整理とタスク管理をどう接続するか

## 現時点の整理

- Web上の情報保存はRaindropが向いている
- 自分用に整理した情報の蓄積はObsidianが向いている
- 両方を横断検索できる仕組みを作りたい
- 日記は1日単位だけでなく、平日・週末など複数日単位の運用も検討したい
- タスク管理はTodoistとNotionで比較検討中
- Todoistは外部操作がしやすそうだが、ツールが増える点が懸念
- Notionは統合しやすそうだが、連携のための準備や実装コストの検証が必要
