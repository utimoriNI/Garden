### Journal daily-reflection 障害対応

### 発生した問題

- Journal の daily-reflection workflow で、`main` への push が non-fast-forward で拒否された。
- 競合対策後、Garden checkout で長すぎるファイル名による `File name too long` が発生した。
- sparse checkout 後、Garden への push で 403 が発生した。
- Garden の checkout は成功しているため、現在の主因は `GARDEN_SYNC_TOKEN` の書き込み権限不足である。

### 実施した対応

- Journal の日次・週次／月次 workflow に concurrency を設定した。
- push 失敗時に fetch・rebase・再 push を行う処理を追加した。
- Garden checkout を sparse checkout に変更した。
- 同期先を `GARDEN_*_PATH_TEMPLATE` から動的に算出するようにした。
- Node 20 の警告は今回の失敗原因ではないことを確認した。

### 現在の状態

- Journal の daily reflection は `main` に登録済み。
- Garden 側の同期は、書き込み権限不足により未完了。
- Garden 用の書き込み可能なトークンを `GARDEN_SYNC_TOKEN` に登録する必要がある。

### 次に行うこと

- `utimoriNI/Garden` に対する Write 権限を確認する。
- Garden を対象にした fine-grained PAT を作成する。
- PAT に `Contents: Read and write` 権限を付与する。
- Journal リポジトリの Actions Secret に `GARDEN_SYNC_TOKEN` として登録する。
- 修正版 `main` から workflow を新規実行する。
