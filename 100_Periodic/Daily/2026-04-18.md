# 日報（2026-04-19）

## 本日の作業概要
LCalibration プロジェクトの初期セットアップと、SwiftUI による iPhone / iPad 向け「同明度トレーニングアプリ」MVP の実装を進めた。

## 実施内容

### 1. Git / GitHub 初期整備
- ローカルディレクトリを Git リポジトリとして初期化
- `main` ブランチを作成
- GitHub に `utimoriNI/LCalibration` リポジトリを新規作成
- `origin` を SSH 接続で設定
- SSH 鍵の生成と接続確認を実施
- 初回コミット `Initial commit` を作成し、`origin/main` へ push

### 2. SwiftUI MVP の実装
- SwiftUI ベースのアプリ構成を新規作成
- `App / Models / Services / Utilities / Storage / ViewModels / Views / Resources` の責務分離構成を採用
- iPhone / iPad の両方を意識したレイアウト方針で画面を構成

### 3. 実装した主要機能
- ホーム画面
  - 学習開始
  - 履歴表示
- モード選択画面
  - モードA: 同明度色の選択問題
  - モードB: 明るさ順ランキング問題
  - モードC: 同明度色の生成問題
  - 問題数選択
  - 難易度選択
- 問題画面
  - 問題番号表示
  - グレースケール表示トグル
  - 各モードに応じた回答 UI
- 問題結果画面
  - 正誤または評価表示
  - ΔY / ランキングスコア表示
  - フィードバック表示
  - 回答時間表示
- セッション結果画面
  - 問題数
  - 正答率
  - 平均ΔY
  - 最大ΔY
  - 平均ランキングスコア
  - 各問題結果一覧
- 履歴画面
  - 端末内保存した過去セッション一覧表示

## ロジック実装内容

### 色計算
- `ColorMath` を作成
- sRGB → linear RGB 変換を実装
- 輝度 `Y = 0.2126R + 0.7152G + 0.0722B` による評価を実装
- `ΔY = |Y_user - Y_target|` を算出可能にした
- HSV / RGB 相互変換を実装

### 問題生成
- `QuestionGenerator` を作成
- モード別に問題を自動生成
- 難易度ごとに候補数・輝度差・生成閾値を調整

### 採点
- モードA
  - 正誤判定
  - 各候補の ΔY 表示
- モードB
  - 正しい順序との順位差をもとにスコア算出
- モードC
  - ΔY と閾値に応じて
    - かなり近い
    - 明るすぎる
    - 暗すぎる
  のフィードバックを返すようにした

### 履歴保存
- `UserDefaults` ベースの簡易履歴保存を実装
- 保存項目
  - 日時
  - モード
  - 問題数
  - 正答率 / スコア
  - 平均ΔY
