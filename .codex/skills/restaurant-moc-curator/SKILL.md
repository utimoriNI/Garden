# 飲食店 MOC 更新スキル

Garden の飲食店 MOC を、投稿や画像から更新するためのスキルです。

## 対象ファイル

必ず次のファイルを更新する。

- `D:\Obsidian\Garden\110_MOC\Living - 飲食店MOC.md`

編集前に対象ファイルを読み、既存の見出し名、インデント、表記ゆれ、重複回避のしかたに合わせる。

## 基本手順

### 1. 入力ソースを集める

次のいずれも入力として扱ってよい。

- X の投稿 URL
- 引用ポストやリプライの URL
- スレッドに添付された画像
- ユーザーが明示したローカル画像パス
- ユーザーが参照したローカルノート

画像が入力の場合は、添付画像を直接読む。ローカル画像パスが渡されたときだけ `view_image` を使う。

### 2. 店名を抽出する

ソース上で確認できる店名だけを抽出する。

抽出時のルール:

- 日本語表記があるならその表記を優先する
- `・` `&` `/` などは元の表記を保つ
- 推測だけで店名を足さない
- 曖昧な候補は書き込む前に保留にする

表記ゆれはまとめる。

例:

- `PST` と `Pizza Studio Tamaki`
- `ベッラ・ナポリ` と `ベッラナポリ`

### 3. エリアとカテゴリを決める

各店について次を決める。

- `## エリア別` のどこに入れるか
- `## カテゴリ別` のどこに入れるか

カテゴリは文脈から明らかな場合に推定してよい。ピザ店の列挙なら `### ピザ` に入れる。

エリアは、対象ファイルですでに使われている見出しを優先する。

- 既存見出しに合うならその見出しを使う
- 既存見出しに無い場合は、ファイル全体の流儀に合う自然なエリア名を新設する
- 新しいエリア見出しを作るときは `### <エリア名>` にそろえる

### 4. Google Maps リンクを解決する

各店に対して、実際に使える Google Maps リンクを付ける。

優先順位:

1. `Living - 飲食店MOC.md` 内にすでにある検証済みリンクを再利用する
2. ブラウズして確認できた Google Maps の place/share リンクを使う
3. place リンクを確認できない場合は、そのまま壊れた place リンクを作らない

重要:

- `https://www.google.com/maps/place/<店名>/` を文字列結合だけで作らない

`maps/place` を書くのは、既存の有効リンクか、実際に確認できた Maps の URL があるときだけにする。確認できない場合は、勝手に壊れたリンクを書かず、未確認としてユーザーに伝える。

### 5. エリア別とカテゴリ別の両方を更新する

各店は必ず次の 2 箇所に反映する。

- `## エリア別`
- `## カテゴリ別`

エリア別では既存パターンに合わせる。

- `- **ピザ**` のようなカテゴリ行を再利用または追加する
- その下に店リンクを並べる

カテゴリ別では該当する `### <カテゴリ名>` の下へ追加する。

リンクテキストは既存の書き方に合わせる。

- 店名だけで書く
- すでにその周辺で `(中目黒)` のようなエリア併記が多ければ合わせる

### 6. 重複を避ける

書き込む前に次を検索する。

- 店名の完全一致
- よくある表記ゆれ
- 同じ Google Maps URL

片方のセクションにだけ存在する場合は、不足している側だけ補う。同じ店を二重に増やさない。

## 編集ルール

- ファイル編集には `apply_patch` を使う
- 変更は必要最小限にとどめる
- 関係ない既存行を書き換えない
- 日本語表記は極力そのまま保つ
- むやみに正規化せず、まずファイルの既存流儀に合わせる

## ユーザーへの返答

作業後は次を短く報告する。

- 追加した店
- 更新した、または新設したエリア見出し
- 更新したカテゴリ見出し
- place リンクを確認できず保留にした店
- 曖昧で見送った店
