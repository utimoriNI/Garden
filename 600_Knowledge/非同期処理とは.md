## 非同期処理とは
- 非同期処理とは**あるタスクの実行中に、他のタスクが別の処理を実行できる**方式
- データベースからのデータの取得などの時間がかかる処理をシングルスレッドで実行すると、実行中は他のタスクを行うことができない
	- 例：DBからのデータ取得処理が起こった後に、ページスクロールをすると、データ取得が終わるまでスクロールが実行されない ^15da7a

##### 同期処理
```mermaid
graph LR

  A[データベースからデータを取得] --> B[スクロール発生] --> C[データ取得完了]--> D[スクロール]
```
##### 非同期処理
```mermaid
graph LR

  A[データベースからデータを取得] --> B[スクロール発生] --> D[スクロール] --> C[データ取得完了] 
```
- シングルスレッドで非同期処理を実現するには、**データ取得などの完了を待たずに次のタスクを実行する**必要がある
- JavaScriptの非同期処理は、処理の完了を待たないという特徴を持つ

## JavaScriptの非同期処理
- JavaScriptの非同期処理はES2015で`Promise`が登場し、ES2017で`async/await`が追加された

### Promiseの基本理解
- Promiseは以下の**3つの状態**がある
	- `pending`：非同期処理の実行中の状態
	- `fulfilled`：非同期処理が正常終了した状態
	- `rejected`：非同期処理が異常終了した状態
- Promiseの基本構文
```js
new Promise(function (resolve, rejet){
	// 非同期処理
})
```
- Promiseのコールバック関数の引数は`resolve`,`reject`が渡る
	- `resolve`：非同期処理が正常終了したことを知らせるメソッド。`return`の代わりに`resolve()`と記述することで、非同期関数が正常終了したことを知らせる
	- `reject`：非同期処理が異常終了したことを知らせるメソッド。`return`の代わりに`reject()`と記述することで、非同期関数が異常終了したことを知らせる
- Promiseオブジェクトを生成するだけだと、`pending`状態のままであるため、非同期処理を呼び出す場合は非同期処理が終了したことを必ず教える必要がある
	- 処理が成功したかを`try,catch`で処理を囲むことで、正常終了・異常終了それぞれの処理を記述できる
```js

new Promise(function (resolve, reject){
  try {
    // 非同期処理
    
    // returnの代わりに正常終了したことを表すresolveを返す
    resolve();
  } catch (e) {
    // 異常終了時の処理
    
    // returnの代わりに異常終了したことを表すrejectを返す
    reject();
  }
})
```

### 非同期処理の実際の動作
```js
new Promise(function (resolve, reject){
  try {
    // 1秒後に"非同期処理"とコンソールに出力
    setTimeout(()=>{
      console.log("非同期処理")
      
      // returnの代わりに正常終了したことを表すresolveを返す
      resolve()
    }, 1000)
  } catch (e) {
    // 異常終了時の処理
    
    // returnの代わりに異常終了したことを表すresolveを返す
    reject()
  }
})
```
- `setTimeout`内で`resolve()`を呼び出している
	- 同期処理であるならば、`setTimeout`で1秒待ってから`resolve()`で正常終了したPromiseが表示される
	- 実際には、`pending`と表示された後に`非同期処理`とログが表示される
	- このように、非同期処理ないで処理の完了を伝える関数を呼んでも、それを待たずに次の処理へ進む

### 非同期処理の完了を待つ
- なぜ非同期処理の完了を待つ必要があるか？
	- ![[非同期処理とは#^15da7a]]
	- 上記の例のようなデータ取得時に終了を待たずに行いたい処理と、データ取得の終了が完了してから実行したい処理が存在するため
		- DBのデータを使用する部分はデータ取得の完了後にレンダリングしたいが、DBのデータ使わないところはデータ取得の完了を待たずにレンダリングしたい
- Promiseに限らず、JavaScriptの非同期処理は戻り値に対して`then`メソッドと`catch`メソッドが利用できる
	- `then()`：非同期処理が正常終了した際に呼ばれるメソッド
	- `catch()`：非同期処理が異常終了した際に呼ばれるソッド
		-  > ローディングのスピナーを表示させる方法がやっと理解できた気がする。データ取得中はスピナーの表示をonにしてthenで表示をoffにするのか
- `then()`, `catch()`はPromiseの状態が変化した時に呼ばれると考えられる
	- `pending`から`resolve`に変化したときに`then()`が呼ばれる
	- `pending`から`reject`に変化したときに`catch()`が呼ばれる

### 非同期で処理した後にその値を取得する
- `then()`と`catch()`はそれぞれ引数を取ることができる
- `resolve()`と`reject()`に引数を渡すことで、`then()`と`catch()`に値を渡すことができる
```js
function asyncFunction() {
  return new Promise((resolve, reject) => {
    try {
      // 1秒後に"非同期処理"とコンソールに出力
      setTimeout(() => {
        console.log("非同期処理")
        const num = 1
        resolve(num);
      }, 1000)
    } catch (e) {
      // 異常終了時の処理

      // returnの代わりに異常終了したことを表すresolveを返す
      reject(e);
    }
  })
}

asyncFunction().then((num) => {
  console.log(`引数で受け取った値：${num}`);
}).catch(e => {
  console.log(`引数で受け取った値：${e}`);
})
```


## async / await
### async / awaitの基本
- ES2017で導入された
- Primiseの糖衣構文（シンタックスシュガー）である
> プログラミング言語において、読み書きのしやすさのために導入される書き方であり、複雑でわかりにくい書き方と全く同じ意味になるものを、よりシンプルでわかりやすい書き方で書くことができるもののこと

#### async
- `async`は非同期関数を作るもの
- 基本構文：`async function asyncFunction() {}`
- 特徴
	- `async function`は呼び出されると`Promise`を返す
	- `async function`が**値を`return`した場合はその値を`resolve`する**
	- `async function`が**例外を`throw`した場合はその値を`reject`する**
- Promiseとの書き方の比較
```js
function promiseFunction() {
  return new Promise((resolve, reject) => {
    try {
      resolve("resolve")
    } catch (e) {
      reject("reject")
    }
  })
}

// アロー関数で書くとこうなります
// const asyncFunction = async () => {
async function asyncFunction() {
  try {
    return "resolve"
  } catch (e) {
    throw "reject"
  }
}

```
- `Promise`が無くなったことで、`resolve`、`reject`が引数に取れなくなり、`return`と`throw`をその代わりとして使っている

#### await
- `await`は非同期処理の終了を待つためのもの
	- `Promise`の結果(`resolve`や`reject`)が返されるまで処理を止める演算子のこと
- **`await`は`async function`の中でしか使えない**
- Promiseとの比較
```js
function promiseFunction() {
  return new Promise((resolve, reject) => {
    try {
      resolve("resolve")
    } catch (e) {
      reject("reject")
    }
  })
}

async function asyncFunction() {
  try {
    return "resolve"
  } catch (e) {
    throw "reject"
  }
}

promiseFunction().then(txt => {
  console.log(txt);
}).catch(e => {
  console.error(e);
})

async function main() {
  const txt = await asyncFunction()
  console.log(txt);
}
main()

```
- `await`は`then()`と`catch()`を置き換えている
	- 非同期関数に対して`await`を使うことで、`resolve()`や`reject()`で返される値を待つことができる

### async / awaitの動作確認でのミス
#### async functionn内のコールバック関数の終了は待ってくれない
```js
async function asyncFunction() {
  try {
    setTimeout(()=>{
      return "resolve"
    }, 1000)
  } catch (e) {
    throw "reject"
  }
}

async function main() {
  const txt = await asyncFunction()
  console.log(txt);
}
main()
```
- async/awaitはPromiseのシンタックスシュガーだが、コンソールに表示されるのは`resolve`ではなく`undefined`
- awaitは非同期処理に対して、`resolve`か`reject`が帰ってくるまで待つが、コールバック関数（ここでは`setTimeout`の第一引数）の終了まで待ってくれない
	- `setTimeout`内の`return`を待たずに`txt`への代入が行われるため、undefinedが出力されてしまう

#### setTimeoutのコールバック関数を非同期関数にしようとする
```js
async function asyncFunction() {
  try {
    setTimeout(async ()=>{
      return "resolve"
    }, 1000)
  } catch (e) {
    throw "reject"
  }
}

async function main() {
  const txt = await asyncFunction()
  console.log(txt);
}
main()
```
- `setTimeout`のコールバック関数を非同期関数にしてその中で`return`をしても、その戻り値である`resolve`は`setTimeout`を呼び出している`asyncFunction`に送られる
	- `asyncFunction`では`resolve`を使っていないので、結果的に`resolve`は消失してしまう

### then, catch, await
- `async/await`は`Promise`のシンタックスシュガーであるため、お互いに利用可能
```js
function promiseFunction() {
  return new Promise((resolve, reject) => {
    try {
      resolve("resolve")
    } catch (e) {
      reject("reject")
    }
  })
}

async function asyncFunction() {
  try {
    return "resolve"
  } catch (e) {
    throw "reject"
  }
}

// promiseFunction() から asyncFunction()に入れ替え
asyncFunction().then(txt => {
  console.log(txt);
}).catch(e => {
  console.error(e);
})

async function main() {
  // asyncFunction() から promiseFunction()に入れ替え
  const txt = await promiseFunction()
  console.log(txt);
}
main()
```
- `async`に対して、`then()`や`catch()`で値を受け取ることができる
- `Promise`に対して、`await`で値を受け取ることができる
